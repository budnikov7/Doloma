# Техническая документация: Авторацион

## Обзор

Модуль "Авторацион" реализует функционал автоматического расчета оптимальных весов кормов в рационе животных на основе ограничений и нормативных показателей питательности.

### Компоненты системы

   - Значения: `Ручной`, `Автоматический`

## Алгоритм оптимизации

### Описание метода

Используется **адаптивный градиентный спуск** с динамическим изменением шага и оценкой качества решения по методу наименьших квадратов отклонений.

### Этапы работы

```
1. Инициализация
   ├─ Проверка входных данных
   ├─ Установка начальных весов (среднее между мин и макс)
   └─ Инициализация параметров оптимизации

2. Итеративная оптимизация (до 1000 итераций)
   ├─ Расчет показателей для текущих весов
   ├─ Оценка качества решения (сумма квадратов отклонений)
   ├─ Сохранение лучшего решения
   ├─ Проверка условия останова (отклонение < 1%)
   └─ Корректировка весов

3. Финализация
   ├─ Применение лучшего найденного решения
   ├─ Проверка показателей на выход за пределы нормы
   └─ Формирование результата
```

### Функция оценки качества

Качество решения оценивается по формуле:

```
Q = Σ (отклонение_i)²

где отклонение_i = {
    (Минимум - Текущее) / Минимум,  если Текущее < Минимум
    (Текущее - Максимум) / Максимум, если Текущее > Максимум
    0,                                иначе
}
```

Чем меньше Q, тем лучше решение.

### Параметры оптимизации

| Параметр | Значение | Описание |
|----------|----------|----------|
| МаксимальноеКоличествоИтераций | 200 | Максимальное число итераций |
| ТочностьСходимости | 0.05 | Критерий останова (5%) |
| НачальныйШаг | 0.3 | Начальный шаг (30% от диапазона) |
| МинимальныйШаг | 0.01 | Минимальный шаг (1% от диапазона) |
| КоэффициентУменьшенияШага | 0.9 | Коэффициент адаптации шага |

## API модуля РасчетАвтоРациона

### Экспортируемые функции

#### РассчитатьАвтоРацион()

```bsl
Функция РассчитатьАвтоРацион(ТаблицаКормов, ТаблицаПитательности, ТаблицаПоказателей, Дата) Экспорт
```

**Параметры:**

- `ТаблицаКормов` (ТаблицаЗначений) - таблица с кормами и ограничениями
  - Колонки: `Номенклатура`, `МинВес`, `МаксВес`
  
- `ТаблицаПитательности` (ТаблицаЗначений) - таблица питательности кормов
  - Колонки: `Номенклатура`, `СухоеВещество`, `ОЭ`, `XP`, `nXp`, `RNB`, и др.
  
- `ТаблицаПоказателей` (ТаблицаЗначений) - нормативные показатели
  - Колонки: `ПоказательРациона`, `ЕдиницаИзмерения`, `Минимум`, `Максимум`
  
- `Дата` (Дата) - дата расчета

**Возвращаемое значение:** Структура

```bsl
{
    "Успех": Булево,              // True если расчет выполнен
    "Сообщение": Строка,          // Текст сообщения
    "РассчитанныеВеса": Соответствие,  // Номенклатура -> Вес
    "КрасныеПоказатели": Массив   // Показатели вне нормы
}
```

### Внутренние функции

1. `ИнициализироватьНачальныеВеса()` - установка начальных значений
2. `РассчитатьПоказателиДляВесов()` - расчет показателей питательности
3. `ОценитьКачествоРешения()` - оценка отклонений от нормы
4. `КорректироватьВеса()` - градиентная корректировка весов
5. `НайтиКрасныеПоказатели()` - поиск показателей вне нормы
6. `НайтиСтрокуПитательности()` - поиск данных по номенклатуре
7. `СкопироватьСоответствие()` - копирование соответствия

## Условное оформление

### Правила выделения

1. **Показатель < Минимум**
   - Цвет: Красный
   - Шрифт: Жирный
   - Условие: `Содержание < Минимум AND Минимум Заполнено`

2. **Показатель > Максимум**
   - Цвет: Красный
   - Шрифт: Жирный
   - Условие: `Содержание > Максимум AND Максимум Заполнено`

## Производительность

### Сложность алгоритма

- **Временная сложность**: O(n × m × k)
  - n - количество кормов
  - m - количество показателей
  - k - количество итераций (до 1000)

- **Пространственная сложность**: O(n + m)

### Оптимизация

1. **Ранний выход** - останов при достижении точности 1%
2. **Кэширование лучшего решения** - сохранение промежуточных результатов
3. **Локальный поиск** - корректировка только одного веса за итерацию

### Типичное время выполнения

- Простой рацион (5-10 кормов): < 1 сек
- Средний рацион (10-15 кормов): 1-3 сек
- Сложный рацион (15-20 кормов): 3-5 сек

## Ограничения и известные проблемы

### Ограничения

1. **Локальный оптимум** - алгоритм может найти локальный, а не глобальный оптимум
2. **Противоречивые ограничения** - если ограничения несовместимы, решение может быть неоптимальным
3. **Фиксированные параметры** - параметры оптимизации жестко заданы в коде

### Известные проблемы

1. **Медленная сходимость** - при большом количестве кормов может потребоваться много итераций
2. **Чувствительность к начальным условиям** - разные начальные веса могут привести к разным решениям

### Планируемые улучшения

1. Реализация симплекс-метода для точного решения
2. Добавление учета стоимости в целевую функцию
3. Настраиваемые параметры оптимизации
4. Многопоточный расчет для больших рационов

## Тестирование

### Тестовые сценарии

1. **Базовый сценарий**
   - 5 кормов с широкими диапазонами
   - 10 показателей с реалистичными нормами
   - Ожидается: успешный расчет без красных показателей

2. **Жесткие ограничения**
   - Узкие диапазоны весов
   - Строгие нормативы
   - Ожидается: решение с несколькими красными показателями

3. **Фиксированные веса**
   - Мин = Макс для всех кормов
   - Ожидается: расчет показателей для заданных весов

4. **Граничные случаи**
   - 1 корм
   - 20+ кормов
   - Отсутствие нормативов
   - Нулевые веса
